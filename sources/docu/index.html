Preliminary Remark

The program was originally written for research purposes without having any user in mind except for the developer. Therefore, the user interface is far from being convenient.

Overview

The program covers everything about contraction hieararchies. Before compilation (just by typing make), make sure that the settings in config.h match your requirements. Below a description of two common use cases:

  • creation of a node ordering
  • creation of a contraction hierarchy

Node Ordering

  • If you only want to create a node ordering in form of a hcn-file.
  • Compile the program with USE_CH_EXPAND (in config.h) switched off. If you use the command line argument "-e" then also with COUNT_SHORTCUT_ORIGINAL_EDGES (in config.h) switched on, otherwise off, in doubt switch it on.  Note: usually an edge needs 8 byte. Each one of those switches adds another 4 byte if switched on.
  • The input graph has to be specified in the ddsg-format. A full example can be found here.
  • Input/Output specification with examples. The class that handles the parameters can be found in command/Sort.h.
  • Use the program to read the original graph and to output a node order in several formats.

 

Node Ordering and Contraction

  • If you want to create a new node ordering and a contraction hierarchy in ch-format.
  • Compile the program with USE_CH_EXPAND (in config.h) switched off. If you use the command line argument "-e" then also with COUNT_SHORTCUT_ORIGINAL_EDGES (in config.h) switched on, otherwise off, in doubt switch it on. Note: usually an edge needs 8 byte. Each one of those switches adds another 4 byte if switched on.
  • Important note: It is possible that you have not enough memory if both of the above switches are on, because every edge will need 16 byte. A solution is to perform the construction in two steps: first create a node ordering in form of a hcn-file (needs 12 byte/node, see above) and then perform a contraction to get the contraction hierarchy in ch-format (needs 12 byte/node, see below).
  • The input graph has to be specified in the ddsg-format. A full example can be found here.
  • Input/Output specification with examples. The class that handles the parameters can be found in command/Sort.h.
  • Use the program to read the original graph and to output a contraction hierarchy and a node order in several formats.

Contraction

  • If you want to create a contraction hierarchy in ch-format and you already have a node ordering as a hcn-file.
  • Compile the program with USE_CH_EXPAND (in config.h) switched on and COUNT_SHORTCUT_ORIGINAL_EDGES  (in config.h) switched off. Note: usually an edge needs 8 byte. Each one of those switches adds another 4 byte if switched on.
  • Make sure that a node ordering has been performed first and a hcn-file is available.
  • Input/Output specification with examples. The class that handles the parameters can be found in command/Contract.h.
  • Use the program to read the node ordering and to create a contraction hierarchy.
