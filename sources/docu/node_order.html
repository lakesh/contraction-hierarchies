Input/Output Specification

The program is completely controlled by command line arguments. Not only the appearance of an argument is important but also the order of the arguments. To create a contraction hierarchy inclusive a node ordering, every call starts with "./main -s -p".

The following arguments are likely to be used:

argument description example
-f graph in ddsg-format -f eur.ddsg
-o output file, the levels in hcn-format, if "-o x" then the file name depends on chosen parameters -o x
-l log file, if "-l x" then the file name depends on chosen parameters -l x
-E export search graph in ddsg-format to given file, if "-E x", then the file name depends on chosen parameters -E x
-Z export search graph to binary sgr-format,  if "-Z x", then the file name depends on chosen parameters -Z x
-C export contraction hierarchy to binary ch-format,  if "-C x", then the file name depends on chosen parameters
Note: to use this parameter, USE_CH_EXPAND (in config.h) needs to be switched on.		 -C x

Additional, the coefficients for the priority function need to be defined along with the limitations to the local searches. However only experienced users should modify the arguments because the description may be misleading or even in some sense wrong. Therefore we propose a predefined set of parameters below in the examples.

argument description example
-x coefficient for the edge difference -x 190
-y coefficient for the search space sizes -y 1
-z coefficient for betweenness -z 400
-b betweenness text file, one line for each node -b eur_betweenness.txt
-r reach text file, one line for each node -r eur_reach.txt
-q coefficient for reach -q 400
-w coefficient for contracted nodes
-v coefficient for new edges
-u coefficient for covering search space
-e coefficient for the sum of original edges of the new shortcuts -e 70
-V coefficient for sqrt(Voronoi regioin) -V 60
-S coefficient for upper bound on settled nodes in query -S 145
-b betweenness text file, one line for each node -b eur_betweenness.txt
-p lazy update trigger, every ? nodes check for complete recalculation -p 1000
-n max settled nodes for weight calculation -n 1000
-m max settled nodes for node contraction -m 1000
-k hop limits, comma-separated list, alternating between hop limit and avg degree limit to switch -k 1,3.3,2,10,3,10,5
-K only used if hop limit active, does update all nodes in hop limit, not only the neighbors, warning: time consuming -K
-L no local edge reduction -L

Examples

economical variant: fast preprocessing, medium space consumption and query times

./main -s -p -f eur.ddsg -o x -l x -x 190 -y 1 -e 600 -p 1000 -k 1,3.3,2,10,3,10,5

 aggressive variant: medium preprocessing, low space consumption and query times

./main -s -p -f eur.ddsg -o x -l x -x 190 -y 1 -p 1000 -n 1000 -V 60 -S 145 -e 70
